Making the canvas application

So we have learnt about how to make classes, event listeners as well as manipulate DOM elements - now we are going to practice combining all these elements and use it to make a canvas application. More specifically, this will function as your base if you decide that you want to work on frontend animation libraries, as many animation libraries such as Three.js and Greensock utilize canvas. This project will also strengthen your problem solving skills - often times, when you want to create something, you have to figure out HOW to do it, and ensuring that you plan out your logic well is quintessential in building an effective and organized application.

Checklist

  • Purpose of project
    • Users will be able to draw on the application, and upload it onto the webpage. This application will function as a way for users to practice self-expression.
  • Minimal Viable Product: Decide what users will be able to do
    • [ ] Users will be able to draw a line
    • [ ] Users will be able to draw a shape (outline and fill)
    • [ ] Users will be able to write text
    • [ ] Users will be able to change the color of the line, shape or text
    • [ ] Users will be able to erase things
    • [ ] Users will be able to get a color using the eyedrop functionality
  • Don't give up on cool features. If we have time...
    • Users will be able to upload images
    • Users will be able to download their images
    • Users will be able to drag things around
    • Users will be able to undo and redo their actions
  • Start here: Canvas Starter Code
    • Please please be sure to practice pulling and pushing before starting. Github branches are a hassle to fix.
    • Look at how the code is structured (look in the examples folder) - more specifically, how are other people creating their application? What are the differences? What is easiest for me to understand? I attached some notes below as to what this process may look like.
      • There's a parent class - mouseMethods, that is extended by the other classes (remember, that means that other classes can utilize the methods in mouseMethods) - you don't necessarily need to use this kind of implementation in designing your application
      • Each button is connected to a class, and we pass in two canvas inside each - why?
      • Make notes for yourself
